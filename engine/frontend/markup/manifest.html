Information

Used to specify a unique identifier for your web application.

The full name of your web application.

May be used when the full name is too long for the available space.

This text helps users understand your app's purpose when viewing it in an app store.

These categories help users discover your app in app stores.

Appearance

This color appears in the application window before your application's stylesheets have loaded.

This color may be applied to various browser UI elements, such as the toolbar, address bar, and status bar.

Determines how much of the browser UI is shown to the user when the app is launched within the context of an operating system.

How the app should be displayed when launched and during use, in relation to the device's screen orientation.

URLs

The URL that should be opened when a user launches your web application.

The top-level URL path that contains your web application's pages and subdirectories.

Shortcuts

Links to key tasks or pages within your web application, typically displayed when a user interacts with the web app's icon.

Icons

Image files that define the icons to represent your web application.

Screenshots

These images help users preview your web app's interface and features in app stores.
